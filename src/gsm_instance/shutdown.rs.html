1//! # Shutdown Module
2//!
3//! This module provides functionality to gracefully shut down a running game server instance.
4//! It sends an interrupt signal to all running server processes (identified by a specific substring in their
5//! executable path) and waits until they have terminated.
6//!
7//! ## Usage
8//!
9//! ```rust,no_run
10//! use gsm_instance::shutdown::blocking_shutdown;
11//!
12//! // Gracefully shut down the server.
13//! // Replace "my_game_server.exe" with the actual executable name.
14//! blocking_shutdown("my_game_server.exe");
15//! ```
16
17use std::{thread, time::Duration};
18use tracing::{debug, info};
19
20use crate::process::ServerProcess;
21
22/// Sends an interrupt signal to all running server processes and waits until they terminate.
23///
24/// This function is designed to perform a graceful shutdown of all processes associated
25/// with a game server instance. It identifies running processes by checking their executable
26/// path against a provided string.
27///
28/// # Arguments
29///
30/// * `executable`: A string slice representing a unique part of the server executable's
31///   path or name. This is used to identify the target processes to shut down.
32///
33/// # Behavior
34///
35/// 1. A `ServerProcess` instance is created to manage process operations.
36/// 2. An interrupt signal (e.g., SIGINT on Unix-like systems) is sent to all processes
37///    whose executable path or name contains the `executable` string.
38/// 3. The function then waits for a short period (5 seconds) to allow processes to begin
39///    their shutdown sequence.
40/// 4. It enters a loop, periodically checking (every 5 seconds) if any matching server
41///    processes are still running.
42/// 5. The loop continues until no matching processes are found, at which point the function
43///    concludes that the server has been successfully stopped.
44///
45/// # Panics
46///
47/// This function does not explicitly panic, but underlying `ServerProcess` operations
48/// might in extreme cases of system resource exhaustion.
49pub fn blocking_shutdown(executable: &str) {
50    let mut server_process = ServerProcess::new();
51    info!("Sending interrupt signal to server processes...");
52    server_process.send_interrupt(executable);
53    // Wait a short while for processes to begin termination.
54    thread::sleep(Duration::from_secs(5));
55    loop {
56        let mut sp = server_process.clone();
57        debug!("Checking if server processes are still running...");
58        if !sp.are_processes_running(executable) {
59            info!("Server processes have been stopped successfully!");
60            break;
61        } else {
62            debug!("Server processes still running. Waiting for 5 seconds...");
63            thread::sleep(Duration::from_secs(5));
64        }
65    }
66}
